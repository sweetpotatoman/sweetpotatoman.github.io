Mosh 最大的特点是基于 UDP 方式传输，一个支持漫游的远程终端程序

# 基本概念

Mosh（mobile shell）是一款基于 UDP 的远程终端软件，包含客户端和服务器两部分，用于代替 SSH。因为 mosh 基于 UDP，所以它可以提供不间断的连接，非常适用于在网络状况不好或时延较大的网络中进行远程终端访问。

mosh 支持方式

  • 创建一个临时的 Key 供客户端一次性连接，退出后失效
  • 通过 SSH 的配置进行认证，但数据传输本身还是自身的 UDP 方式

# 环境

本文使用 ssh 的配置进行连接目标服务器

  • 服务器系统：ubuntu 18.04 TLS
  • 客户端系统：mac 10.14.5

# 安装

客户端和服务器上都需要安装 mosh

  1. ubuntu下安装 mosh 服务器

    sudo apt-get install mosh

  2. mac 下安装 mosh 服务器

    brew install mosh

# 端口开放

开放 60000-61000 的 UDP 端口，或者关闭服务器防火墙服务
(如果是云服务器，调整安全组策略即可)

  • iptables

    sudo iptables -I INPUT 1 -p udp --dport 60000:61000 -j ACCEPT

  • ufw

    sudo ufw allow 60000:61000/udp

  • firewalld

    # 先确定活动区域
[root@one ~]# firewall-cmd --get-active-zones
public
  interfaces: eth0

# 编辑防火墙规则
[root@one ~]# firewall-cmd --permanent --zone=public   --add-port=60000-61000/udp

# 让防火墙生效
[root@one ~]# firewall-cmd --reload

# example

连接 fat1 服务器

# mac 终端
~ > mosh fat1

# 连接至 fat1 中
ubuntu@ip-10-0-11-99:~$

可以看出 fat1 中有 mosh 进程且占用默认分配的 60001 端口

root@ip-10-0-11-99:/home/ubuntu# ps -ef|grep mosh
ubuntu   25096     1  0 02:47 ?        00:00:00 mosh-server new -c   256 -s -l LANG=zh_CN.UTF-8
root     25787 10928  0 02:49 pts/0    00:00:00 grep --color=auto   mosh

root@ip-10-0-11-99:/home/ubuntu# netstat -anp|grep 25096
udp        0      0 10.0.11.99:60001        0.0.0.0:*                           25096/mosh-server
